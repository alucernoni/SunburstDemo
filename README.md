# Congressional Stock Trading — Sunburst Visualization

An interactive D3.js sunburst chart exploring stock trades disclosed by members of the U.S. Congress. Each arc represents a party, politician, or stock ticker — sized by trading volume and colored by alpha performance relative to the S&P 500.

---

## What it shows

The chart has three concentric rings:

| Ring | Represents | Sized by | Colored by |
|---|---|---|---|
| Inner | Party | Trading volume | Party (blue / orange / purple) |
| Middle | Politician | Trading volume | Alpha vs S&P 500 (green → gray → red) |
| Outer | Stock ticker | Trading volume | Parent politician's alpha (lighter) |

**Alpha** is calculated on purchase transactions only — sales can be non-discretionary, purchases are deliberate bets. Each purchase is benchmarked against SPY over the same 1-year holding window. Politician alpha is volume-weighted across all their purchases.

Only legislators with 10+ disclosed purchase transactions are included (~134 total).

---

## Interaction

| Action | Result |
|---|---|
| Click a **party arc** | Zoom in — expands party to full circle |
| Click a **politician arc** | Zoom in — expands politician to full circle, shows all tickers |
| Click **N others** (politicians) | Expands the collapsed small politicians in place; auto-zooms party for readable labels |
| Click **N others / N tickers** (tickers) | Opens a side panel listing all tickers with alpha and volume |
| Click anywhere already zoomed | Zoom out one level |
| Hover any arc | Tooltip with full stats |
| **Min Alpha slider** | Filters to politicians at or above the threshold |
| **Current only** toggle | Filters to currently-serving legislators |

## Tech stack

- **Frontend:** React 19 + TypeScript, Vite
- **Visualization:** D3 v7 — "D3 owns the DOM" pattern (`useRef` on `<svg>`, all rendering in `useEffect`)
- **Data pipeline:** Python 3 (pandas, yfinance)
- **Font:** [Inter](https://fonts.google.com/specimen/Inter)

## Getting started

### 1. Install frontend dependencies

```bash
npm install
```

### 2. Run the data pipeline

The pipeline fetches price data from yfinance, calculates alpha, and builds `public/hierarchy.json`.

```bash
cd pipeline
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install pandas yfinance
cd ..
python run_pipeline.py
```

Skip the slow fetch steps if prices are already cached:

```bash
python run_pipeline.py --skip-fetch
```

Required input files in `pipeline/data/raw/`:

| File | Source |
|---|---|
| `congress-trading-all.csv` | [Kaggle — Congressional Trading](https://www.kaggle.com/datasets/shabbarank/congressional-trading-inception-to-march-23?resource=download) |
| `legislators-current.csv` | [unitedstates/congress-legislators](https://github.com/unitedstates/congress-legislators) |

### 3. Start the dev server

```bash
npm run dev
```

Open `http://localhost:5173`.

> Browsers block `fetch()` on `file://` — the dev server is required.

## Crowding strategy

Congress has hundreds of legislators each trading dozens of tickers — naive rendering produces unreadable slivers. The app handles this at two levels:

**Politicians:** the bottom 10% by volume within each party (or any excess beyond 24 visible) are collapsed into a single "N others" arc. Clicking it swaps them into view and auto-zooms the party ring so every small politician gets enough arc for readable labels.

**Tickers:** the app calculates exactly how many ticker arcs fit with readable labels at the current chart size, and collapses the rest into a "N tickers" arc. This recalculates on every resize, filter change, and zoom — so labels are always present or absent, never just illegibly small.
